Week 1 | Getting Started

Pre-Week1: Before the official start of Codefest, we formed our group and communicated on WhatsApp where we decided our project concept: an educational game + website about the pandemic.
In the first week, we then started working out details, game concepts, art style, and collected our ideas on Google Doc. Later, we switched over to Discord so that we could more easilly share images and group call. A website framework which the game is intended to be hosted on is created pushed onto Github.

Preparation:

  • Set up a UnityID, UnityHub, and Unity Version 2019.4.17f1
  • Look for and/or create assets
  • Look up and follow online tutorials

Our Intial Ideas:

  • Game about epidemics and contact tracing
  • Target Audience: General Public
  • Point and Click Style game
  • First Person Perspective
  • Need concrete objectives
  • End Goal: Limit the spread of the virus using Covid-19 data
  • The player clicks on textboxes to make a decision
  • When the player makes a choice, there will be a popup that will explain why the selected choice was good or bad
  • The player's choices affect future events
  • The player can play as a person in power

Week 6

The Unity Scripts

  • TabChanger.cs
  • InventoryUpdater.cs
  • MenuControl.cs
  • PanelUpdater.cs
  • KitchenActions.cs
  • KitchenProcessControl.cs
  • Bed.cs

    • Scripts that activate, deactivate, and modify game objects and what they contain under certain conditions while the game is running. A summary of what these scripts do:

    • Activate and deactivate game objects like prompts, menus and images when conditions are met.
    • Make buttons interactable or not based on the values of certain variables.
    • Change text contained in buttons, images or panels.
    • Modify the game's static variables in response to player actions.
  • Tooltip.cs
  • TooltipSystem.cs
  • TooltipTrigger.cs

    • Scripts that allow attaching tooltips that show when hovering over objects in scenes.

    • Tooltip.cs sets text fields, manages how the tooltip resizes (upon reaching a certain number of characters), and sets the position of the tooltip relative to the position of the mouse pointer which changes based on the scenes' canvas render mode.
    • TooltipSystem.cs creates the Tooltip object and contains the tooltip's show and hide methods.
    • TooltipTrigger.cs activates a tooltip when the mouse pointer is hovered over a game object it is attached to. It is also where the content of the text fields inside the tooltip is determined.
  • Game.cs

    • The main essential game management and control script. It creates all the static data variables that are utilized within the game, methods that allow for modifying them, and methods to create time-based or other types of processes to run in the background when the game starts. Some time-based processes include coffee brewing and stove cooking which take a certain amount of time to complete and for their changes to take effect in the game. The script also contains methods that allow for saving and loading the game at any point or state.

  • GameData.cs
  • SaveSystem.cs
    • Main components of the game's saving/loading system. Upon creating a GameData object, all static game data in Game.cs is retrieved and saved in the object.
    • In SaveSystem.cs, SaveGame() method utilizes binary serialization to create a secure game data file and then saves it in Unity's persistent data path (changes based on platform).
    • BinaryFormatter formatter -> Binary formatting object.
      FileStream stream -> Create a file at path.
      formatter.Serialize(<Filestream>,<data>)
  • LoadGame() method then allows for loading game data from the file saved in the file path if such a file exists.

Reflection

Looking back on the past month of working on this project, we feel like we learned a lot. Many of us had no experience with Unity and game development. Arifa and Ahmed were the most experienced in the group so they lead the scripting in Unity. However, regardless of experience, we use used tutorials and forum to aid us in coding the game and website. During the first and second week, we mostly brainstormed ideas in a Google Doc. We kept communicating through the groupchat and group calls to report finished task, new features, and new ideas. In the Following week, we refined our ideas so it was detailed enough to code it. For example, there was some confusion about how buying food would look like and how the information is used. We decided that the player would see a list of items in the supermarket and when they go to the kitchen, the items added to a variable. If the player bought a healthy food, then the variable represent the healthy food supply is increased. This method would eliminate the need for an inventory bar and the need for drawing specific foods. While deciding how features are implemented, we had to consider our time constraint and the difficulty of implementing a feature. During the last week, we tried to finsh all the idea we talked about. If we had more time, we would like to implement a feature that allows the player to get infected and get tested for Covid-19. We thought that the game wouldn't be enough to provide infomation about Covid-19 so we created a website so we can provide more infomation about it.
